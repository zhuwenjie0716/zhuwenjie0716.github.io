tcpdump安装与使用

Tcpdump

tcpdump 是什么

1988 年，劳伦斯伯克利国家实验室的四位工程师编写出了 tcpdump 这个殿堂级的工具。它可以工作在各种 Unix 类的操作系统上，包括 Linux、FreeBSD、macOS、Solaris 等，也是目前使用最为广泛的抓包工具之一。不过 tcpdump 要过滤报文的话，还要依赖一个底层能力：BPF。

BPF 全称是 Berkeley Packet Filter（也叫 BSD Packet Filter），它是 tcpdump 等抓包工具的底层基础。在 BPF 出现之前，虽然各家操作系统都有自己的抓包工具，但也都有这样或那样的不足。BPF 以一种新的基于寄存器的虚拟机方式，实现了高效稳定的报文过滤功能。

BPF 实现了抓包虚拟机，但用户态的程序是没法直接使用的。于是，libpcap 出现了，它提供了 API 给用户空间程序（包括 tcpdump、Wireshark 等），使得后者能方便地调用 BPF 实现抓包过滤等功能。

也就是说，当使用 tcpdump 时，其实是走了 tcpdump -> libpcap -> BPF 的逻辑。

离线安装

以下需要的安装包可以从 Linux离线安装抓包工具Tcpdump全流程（从零基础到精通） 里找到下载地址。

  1. 安装 gcc

  2. 安装 m4、flex、bison 等工具

  3. 安装 libpcap

  4. 安装 tcpdump

使用 Tcpdump

Tcpdump 用于 linux 对网络进行抓包，可将抓包结果导出用 wireshark 查看

过滤语法

基于 IP 地址进行过滤 host

基于网段进行过滤 net

基于端口进行过滤，port，支持【src】【dst】

基于协议进行过滤 protocal

参数语法

指定监听网络设备 -i

将抓包结果写入文件 -w，从文件中读取 -r

不解析 IP 地址和端口号 -n,-nn,-N

指定打印的时间格式 -t

输出更详细的抓包信息 -v

指定抓包条数 -c

指定抓包大小 -C，与 -W 配合使用，文件 -C 大小后就写一个新文件，最多写 -W 个，-C 单位百万字节

指定包的方向 -Q

简洁打印信息 -q

查看本机网络接口信息 -D, -L

指定每个包捕获的长度 -s

以 ASICC 码格式打印包内容信息，方便 grep 进行分析 -A

以 16 进制格式打印包内容信息 -x

打印数据链路层内容信息 -e

以某文件中的规则进行抓包 -F，例如某些包抓一次要好多参数，就可以配置为文件使用

同时输出到终端和文件 -l

基于包大小进行过滤 less / greater，单位 byte

除此之外 tcpdump 也支持一些逻辑符号,例如抓取端口不是 80 上的数据
